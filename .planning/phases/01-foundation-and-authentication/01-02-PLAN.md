  <action>
    1. Create `src/lib/auth/auth-options.ts` following research Pattern 1:
       - Import NextAuth from "next-auth", Credentials from "next-auth/providers/credentials", PrismaAdapter from "@auth/prisma-adapter"
       - Configure Credentials provider with authorize function that:
         a. Validates input with signInSchema.safeParse()
         b. Finds user by email via prisma.user.findUnique()
         c. Compares password with bcryptjs.compare() against user.passwordHash
         d. Returns {id, email, name, image} on success, null on failure
       - Use JWT session strategy (NOT database strategy -- Auth.js v5 Credentials provider does not work with database sessions because authorize() returns User but adapter expects Account linkage. JWT strategy is the correct approach for Credentials).
       - Set session maxAge to 30 days (30 * 24 * 60 * 60)
       - Configure callbacks:
         - jwt callback: attach user.id to token on first sign-in
         - session callback: copy token.sub (user id) to session.user.id
       - Configure custom pages: signIn: "/login"
       - Export: auth, handlers (GET, POST), signIn, signOut

    2. Create `src/lib/auth/get-current-user.ts`:
       ```typescript
       import { auth } from "./auth-options";
       export async function getCurrentUser() {
         const session = await auth();
         if (!session?.user?.id) return null;
         return session.user;
       }
       ```

    3. Create `src/app/api/auth/[...nextauth]/route.ts`:
       ```typescript
       import { handlers } from "@/lib/auth/auth-options";
       export const { GET, POST } = handlers;
       ```

    4. Create `src/app/api/auth/signup/route.ts` as a POST endpoint:
       - Parse request body with signUpSchema.safeParse()
       - Return 400 with validation errors if invalid
       - Check if email already exists (prisma.user.findUnique)
       - Return 409 if email taken
       - Hash password with bcryptjs.hash(password, 13) -- cost factor 13 per OWASP
       - Create user with prisma.user.create({email, passwordHash, name: displayName})
       - Return 201 with {id, email, name} (never return passwordHash)

    5. Create `middleware.ts` at project root:
       - Export auth from auth-options as middleware (Auth.js v5 pattern)
       - Or use NextRequest-based middleware that checks for session token cookie
       - Protected routes: /dashboard and /dashboard/* paths
       - Redirect unauthenticated users to /login
       - Allow /login, /signup, /forgot-password, /reset-password, /api/auth/* without auth
       - Configure matcher: ["/dashboard/:path*"]

    6. Create `src/types/next-auth.d.ts` to extend the Session type:
       ```typescript
       import { DefaultSession } from "next-auth";
       declare module "next-auth" {
         interface Session {
           user: { id: string } & DefaultSession["user"];
         }
       }
       ```
