  <name>Task 1: Implement password reset flow (forgot-password and reset-password)</name>
  <files>
    src/app/(auth)/forgot-password/page.tsx, src/app/(auth)/reset-password/page.tsx,
    src/app/api/auth/forgot-password/route.ts, src/app/api/auth/reset-password/route.ts
  </files>
  <action>
    1. Create `src/app/api/auth/forgot-password/route.ts` following research Pattern 3:
       - POST endpoint accepting {email}
       - Validate with forgotPasswordSchema
       - Look up user by email -- if not found, still return success (don't reveal if email exists)
       - Generate 32-byte random token via crypto.randomBytes(32).toString("hex")
       - Hash token with SHA-256 before storing (crypto.createHash("sha256").update(token).digest("hex"))
       - Store in prisma.verificationToken.create: identifier=email, token=hashedToken, expires=now+1hour
       - Console.log the reset link: `[DEV] Password reset link for ${email}: ${NEXTAUTH_URL}/reset-password?token=${rawToken}`
       - Return {success: true} regardless of whether email exists (security)

    2. Create `src/app/api/auth/reset-password/route.ts`:
       - POST endpoint accepting {token, password}
       - Validate with resetPasswordSchema
       - Hash the incoming token with SHA-256 (same as generation)
       - Look up verificationToken by hashed token
       - If not found or expired (expires < new Date()): return 400 "Token expired or invalid"
       - Hash new password with bcryptjs.hash(password, 13)
       - Use prisma.$transaction to atomically:
         a. Update user passwordHash
         b. Delete the verificationToken
       - Return {success: true}

    3. Create `src/app/(auth)/forgot-password/page.tsx`:
       - Client component with email input form
       - Use Input component from Plan 03
       - On submit: POST to /api/auth/forgot-password
       - Show success message: "If an account exists with that email, we've sent a reset link. Check your server console in development."
       - Always show success (even if email doesn't exist) to prevent email enumeration
       - Link back to /login
       - Style consistent with login/signup pages (auth layout)

    4. Create `src/app/(auth)/reset-password/page.tsx`:
       - Client component that reads token from URL search params (useSearchParams)
       - If no token in URL: show error "Invalid reset link"
       - New password input with strength requirements displayed
       - Confirm password input (client-side match validation)
       - On submit: POST to /api/auth/reset-password with {token, password}
       - On success: show "Password reset successfully" with link to /login
       - On error (expired/invalid): show error with link to /forgot-password to request new link
       - Client-side validation: passwordSchema on new password, match confirmation
  </action>
