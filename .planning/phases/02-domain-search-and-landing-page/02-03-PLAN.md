1. **Create `src/app/api/domain/check/route.ts`** (Route Handler):
   - Import `NextRequest`, `NextResponse` from `next/server`
   - Import `checkDomains` from `@/lib/rdap/rdap-client`
   - Import `domainSearchSchema` from `@/lib/validators/domain.schema`
   - Import `generateSuggestions` from `@/lib/suggestions`
   - Export `async function POST(req: NextRequest)`:
     - Parse body with `req.json()`
     - Validate with `domainSearchSchema.safeParse(body)` -- return 400 with error messages on failure
     - Call `checkDomains(data.domains)`
     - For taken domains, generate suggestions: extract domain name (before first dot), collect taken TLDs, call `generateSuggestions(name, takenTlds)`
     - Return `NextResponse.json({ results, suggestions })` with 200
     - Wrap in try/catch, return 500 on unexpected errors
   - Do NOT export GET -- search is POST-only (sends domain list in body)

2. **Create `src/components/domain/search-results.tsx`** ("use client"):
   - Import `DomainCheckResult` type from `@/lib/rdap/rdap-client` (type-only import)
   - Props: `{ results: DomainCheckResult[], suggestions: string[], isLoading: boolean, onSearchSuggestion: (domain: string) => void }`
   - Render each result as a row (flex, justify-between, items-center):
     - Left: domain name in mono font (font-[family-name:var(--font-jetbrains-mono)] or font-mono), bold
     - Center: price from result.price
     - Right: availability badge
       - available=true: green badge "Available" (bg-green/10 text-green) + "Register" button (bg-fire text-black rounded-full)
       - available=false: red badge "Taken" (bg-red/10 text-red) + "Make Offer" button (bg-surface text-text2 border border-border rounded-full)
       - available='unknown': gold badge "Unknown" (bg-gold/10 text-gold)
       - isLoading state: gold badge "Checking..." with spinner animation
   - Row styling: bg-bg2 border border-border rounded-[--radius-sm] px-[22px] py-[14px] mb-1.5, fadeUp animation
   - If `suggestions.length > 0` and any results are taken, show "Suggestions" section below results with suggestion chips that call `onSearchSuggestion`
   - Handle empty state: if results is empty and not loading, show nothing

3. **Create `src/components/domain/search-bar.tsx`** ("use client"):
   - Import `searchInputSchema` from `@/lib/validators/domain.schema`
   - Import `DEFAULT_TLDS` from `@/lib/rdap/tld-config`
   - Import `DomainCheckResult` type and `SearchResults` component
   - Import `BulkSearch` component
   - State: `query`, `error`, `results: DomainCheckResult[]`, `suggestions: string[]`, `isLoading`, `isBulkMode`
   - Search bar UI matching index.html: rounded-full border-2 border-border bg-bg2, focus-within:border-fire with shadow glow
   - Input: flex-1 bg-transparent px-6 py-4, font-outfit, placeholder "Find your perfect domain name..."
   - Button: bg-gradient-to-r from-fire to-gold px-8 py-4 font-anybody font-bold text-black
   - TLD pills below: row of `.com`, `.io`, `.ai`, `.app`, `.se`, `.nu`, `.tech`, `.xyz`, `.dev` buttons that append to query
   - Bulk search toggle: checkbox + label "Bulk domain search", toggles `isBulkMode`
   - **Search logic:**
     - Validate input with `searchInputSchema.safeParse()` -- show error message if invalid, do NOT search
     - If input has TLD (contains `.` and last segment >= 2 chars), search that exact domain
     - If no TLD, expand to all DEFAULT_TLDS: `DEFAULT_TLDS.map(t => '${name}.${t}')`
     - POST to `/api/domain/check` with `{ domains: [...] }`
     - Show placeholders immediately (results with checking status)
     - Update results on response
   - **Enter key** triggers search, button click triggers search
   - Render `<SearchResults>` below search bar when results exist
   - Render `<BulkSearch>` when `isBulkMode` is true (pass shared search handler)

4. **Create `src/components/domain/bulk-search.tsx`** ("use client"):
   - Props: `{ onSearch: (domains: string[]) => void }`
   - Textarea: bg-bg2 border border-border rounded-[--radius-sm] p-3.5, font-mono text-[13px], min-h-[100px], resize-vertical
   - Placeholder: "Enter one domain per line, e.g.\ntaco-shop.com\nsalsa-hub.io\nguac-lord.dev"
   - "Search All" button (btn-fire style) that:
     - Splits textarea by newlines, trims, filters empty, lowercases
     - Validates each with searchInputSchema -- skips invalid, shows count of skipped
     - Calls `onSearch` with valid domain list (max 50)
   - Show validation summary: "Searching N domains" or "N invalid domains skipped"
