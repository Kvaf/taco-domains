3. **Create `src/lib/rdap/tld-config.ts`** exactly as specified in RESEARCH.md Pattern 2:
   - Export `TLDConfig` interface with `{ tld, rdapServer, price, label }`
   - Export `TLD_CONFIGS` array with all 14 TLDs (com, net, org, io, ai, app, dev, xyz, tech, se, nu, info, co, taco)
   - Prices must match existing index.html: com=$12.99, io=$39.99, ai=$79.99, taco=$4.20, etc.

   - Export `TLD_MAP` as `Record<string, TLDConfig>` from `Object.fromEntries`
   - Export `DEFAULT_TLDS` array: `['com', 'io', 'ai', 'app', 'se', 'nu', 'tech', 'xyz', 'dev', 'taco']`

4. **Create `src/lib/validators/domain.schema.ts`** using Zod v4 (`import { z } from 'zod/v4'`):
   - `domainLabelSchema`: validates a single label (before TLD). Rules: 1-63 chars, must start and end with alphanumeric, only lowercase letters/numbers/hyphens, hyphens in positions 3-4 forbidden unless `xn--` prefix (IDN). Use `.min(1)`, `.max(63)`, `.regex()` for start/end, `.refine()` for positions 3-4 check.
   - `fullDomainSchema`: validates complete domain (label.tld). Max 253 chars, `.transform(val => val.toLowerCase().trim())`, then pipe to regex for `label.tld` format.
   - `domainSearchSchema`: `z.object({ domains: z.array(z.string()).min(1).max(50) })`
   - `searchInputSchema`: validates raw user input before TLD expansion. Min 1, max 253, transform lowercase+trim, refine for valid chars.

5. **Create `src/lib/suggestions.ts`** as specified in RESEARCH.md:
   - Export `generateSuggestions(name: string, takenTlds: string[]): string[]`
   - Strategy: (a) Try other DEFAULT_TLDS not in takenTlds, (b) prefix variations (my-, get-, try-, use-, go-) for .com, (c) suffix variations (-app, -hq, -hub) for .com, (d) hyphen toggling (remove if present, add at midpoint if absent and name > 6 chars)
   - Deduplicate with `Set`, return max 8 suggestions

6. **Write tests:**
   - `src/lib/rdap/tld-config.test.ts`: verify TLD_CONFIGS has 14 entries, all have required fields, TLD_MAP lookup works, DEFAULT_TLDS has 10 entries, prices match expected values for key TLDs (.com=12.99, .taco=4.20)
   - `src/lib/validators/domain.schema.test.ts`: test valid domains ("taco-shop", "a", "xn--nxasmq6b"), test invalid domains ("", "-start", "end-", "ab--cd" non-xn, 64-char label, chars like "test!"), verify specific error messages
   - `src/lib/suggestions.test.ts`: test generates suggestions array, respects takenTlds exclusion, returns max 8, includes TLD alternatives and name variations, handles hyphenated names

<task type="auto" tdd="true">
  <name>Task 2: RDAP client with DNS fallback and rate limiting</name>
  <files>src/lib/rdap/rdap-client.ts, src/lib/rdap/dns-fallback.ts, src/lib/rdap/rdap-client.test.ts</files>
  <behavior>
    - checkSingleDomain("test.taco") returns available=true, source="simulated" (always available)
    - checkSingleDomain("test.com") with RDAP 404 returns available=true, source="rdap"
    - checkSingleDomain("google.com") with RDAP 200 + domain data returns available=false, source="rdap"
    - checkSingleDomain("test.com") with RDAP timeout falls back to DNS, source="dns"
    - checkSingleDomain with unknown TLD (no RDAP server) returns available="unknown"
    - checkDomains processes multiple domains in parallel via Promise.all
    - dnsFallback returns available=true on NXDOMAIN (Status=3), available=false on Answer present
    - Rate limiting: p-queue with concurrency=3 per RDAP server prevents IP bans
  </behavior>
  <action>
1. **Create `src/lib/rdap/dns-fallback.ts`:**
   - Export `async function dnsFallback(domain: string, tld: string, price: string): Promise<DomainCheckResult>`
   - Call `https://dns.google/resolve?name=${domain}&type=A` with `AbortSignal.timeout(4000)`
   - Status 3 (NXDOMAIN) = available=true, Answer array present = available=false, otherwise unknown
   - All return `source: 'dns'`
   - Wrap in try/catch, return unknown on any failure
   - Import `DomainCheckResult` type from `./rdap-client` -- use a separate types file or co-locate the type

2. **Create `src/lib/rdap/rdap-client.ts`:**
   - Import `PQueue` from `p-queue`, `TLD_MAP` from `./tld-config`, `dnsFallback` from `./dns-fallback`
   - Define and export `DomainCheckResult` interface: `{ domain, tld, available: boolean | 'unknown', price: string, source: 'rdap' | 'dns' | 'local' | 'simulated' }`
   - Constants: `RDAP_TIMEOUT = 6000`
   - Queue management: `Map<string, PQueue>` keyed by RDAP server URL. `getQueue(server)` creates with `{ concurrency: 3, interval: 1000, intervalCap: 5 }`
   - `checkSingleDomain(domain: string)`: extract TLD, lookup config, handle .taco (always available, source=simulated), handle null rdapServer (unknown), queue the RDAP fetch, parse response (404/400=available, 200 with objectClassName/ldhName=taken), catch to dnsFallback
   - `checkDomains(domains: string[])`: `Promise.all(domains.map(checkSingleDomain))`
   - RDAP URL construction: if server starts with `https://rdap.org` use `https://rdap.org/domain/${domain}`, otherwise append domain to server URL

3. **Write `src/lib/rdap/rdap-client.test.ts`:**
   - Mock global `fetch` using `vi.fn()` to control RDAP and DNS responses
   - Test .taco always returns available/simulated without fetch calls
   - Test RDAP 404 returns available
   - Test RDAP 200 with `{ objectClassName: 'domain', ldhName: 'test.com' }` returns taken
   - Test RDAP failure triggers DNS fallback
   - Test unknown TLD with no rdapServer config
   - Test checkDomains with multiple domains returns array
   - Mock PQueue or use real PQueue (tests should run fast since fetch is mocked)
  </action>
